## 二、你需要做的具体事情（按顺序）

### 任务 1：搭建 MySQL + Sequelize 环境

#### 1.1 安装依赖

在 `server` 目录下执行：

```bash
npm install sequelize mysql2 bcrypt cors
```

- **sequelize**：ORM，用 JS 操作 MySQL（定义模型、增删改查）
- **mysql2**：MySQL 驱动，Sequelize 依赖它连接数据库
- **bcrypt**：密码加密库，注册时加密、登录时校验
- **cors**：跨域支持，前端调用接口时不会报错（可选，但建议先装上）

#### 1.2 确保 MySQL 已启动

- 在终端执行 `mysql -u root -p`，能进入说明 MySQL 已运行
- 若未安装，按「学习与实施计划」第一天要求先装好 MySQL

#### 1.3 创建数据库

在 MySQL 中执行：

```sql
CREATE DATABASE my_mall DEFAULT CHARACTER SET utf8mb4;
```

### 任务 2：设计并实现 Sequelize 模型

#### 2.1 新建 Sequelize 配置与连接

在 `server` 下新建 `config/db.js`（或 `db.js`），写入数据库连接配置：

- 主机、端口、数据库名、用户名、密码
- 用 `Sequelize` 实例连接，并导出

#### 2.2 创建 `models` 目录，定义模型

**建议先做 2 个：User、Goods**。其余（Cart、Address、Order）可 Day 2～3 末期或第 4 天再加。

| 模型 | 主要字段（示例） | 说明 |
|------|------------------|------|
| **User** | id, username, password, email, phone, createdAt, updatedAt | 密码字段存 bcrypt 加密后的值 |
| **Goods** | id, name, price, image, description, stock, createdAt, updatedAt | 商品名、价格、图片、描述、库存 |

**你需要做的事**：
1. 新建 `server/models/User.js`、`server/models/Goods.js`
2. 用 `sequelize.define()` 定义字段、类型、是否必填
3. 新建 `server/models/index.js` 统一导出各模型，并调用 `sequelize.sync()` 同步到数据库

**可参考结构**：
- `User`：username、password 必填，email、phone 可选
- `Goods`：name、price 必填，image、description、stock 可选

### 任务 3：用户注册接口

#### 3.1 接口设计

- **URL**：`POST /api/user/register`
- **请求体**：`{ username, password, email? }`
- **成功返回**：`{ code: 0, message: '注册成功', data: { id, username } }`
- **失败返回**：`{ code: 1, message: '用户名已存在' }` 等

#### 3.2 你需要做的事

1. 引入 `bcrypt`，对 `password` 做 `bcrypt.hash(password, 10)`
2. 用 `User.create({ username, password: 加密后, email })` 插入
3. 若 `username` 已存在，先 `User.findOne({ where: { username } })` 判断，存在则返回失败
4. 在 `app.js` 或 `routes/user.js` 中挂载该路由

---

### 任务 4：用户登录接口

#### 4.1 接口设计

- **URL**：`POST /api/user/login`
- **请求体**：`{ username, password }`
- **成功返回**：`{ code: 0, message: '登录成功', data: { id, username, token? } }`（token 可先不实现，只返回用户信息）
- **失败返回**：`{ code: 1, message: '用户名或密码错误' }`

#### 4.2 你需要做的事

1. 用 `User.findOne({ where: { username } })` 查用户
2. 若不存在，直接返回「用户名或密码错误」
3. 若存在，用 `bcrypt.compare(password, user.password)` 校验密码
4. 校验通过返回用户信息；失败返回「用户名或密码错误」
5. **不要**在返回里带上 `password` 字段（可用 `{ attributes: { exclude: ['password'] } }` 或手动删掉）

---

### 任务 5：商品列表接口

#### 5.1 接口设计

- **URL**：`GET /api/goods/list`
- **可选参数**：`page`、`pageSize`（分页，可先不做）
- **成功返回**：`{ code: 0, data: [ { id, name, price, image, ... }, ... ] }`

#### 5.2 你需要做的事

1. 用 `Goods.findAll()` 查全部商品
2. 可先**写死几条数据**：在 `models/index.js` 的 `sync` 后，用 `Goods.bulkCreate([...])` 插入 3～5 条测试数据（或单独写一个初始化脚本）
3. 接口里 `Goods.findAll()` 后 `res.json` 返回

---

### 任务 6：商品详情接口

#### 6.1 接口设计

- **URL**：`GET /api/goods/:id`
- **成功返回**：`{ code: 0, data: { id, name, price, image, description, stock, ... } }`
- **失败返回**：`{ code: 1, message: '商品不存在' }`

#### 6.2 你需要做的事

1. 用 `Goods.findByPk(req.params.id)` 查商品
2. 若不存在，返回「商品不存在」
3. 若存在，返回商品详情

## 四、用 Postman 验收（必做）

做完以上接口后，按下面步骤在 Postman 里测一遍。

| 接口 | 方法 | URL | 预期 |
|------|------|-----|------|
| 注册 | POST | `http://localhost:3000/api/user/register` | Body 传 `{ "username": "test", "password": "123456" }`，返回成功 |
| 登录 | POST | `http://localhost:3000/api/user/login` | Body 传 `{ "username": "test", "password": "123456" }`，返回用户信息 |
| 商品列表 | GET | `http://localhost:3000/api/goods/list` | 返回商品数组 |
| 商品详情 | GET | `http://localhost:3000/api/goods/1` | 返回 id=1 的商品 |

**注意**：
- POST 请求在 Postman 里选 **Body → raw → JSON**
- Content-Type 会自动设为 `application/json`（或手动选）
