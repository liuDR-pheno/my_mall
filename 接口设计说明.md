# 接口是怎么设计出来的

> 结合项目实际代码，说明一个接口从设计到实现的过程，包括语法、函数和注意事项。

---

## 一、设计流程概览

一个接口从想法到落地，大致三步：

1. **定功能**：这个接口要干嘛？（注册、登录、查商品？）
2. **定协议**：用啥 HTTP 方法、路径、传什么参数、返回什么
3. **写代码**：用 Express 把逻辑实现出来

---

## 二、基本语法

### 1. 路由注册方式

```javascript
// 格式：router.方法(路径, 处理函数)

router.get('/list', (req, res) => { ... });      // GET 请求
router.post('/register', (req, res) => { ... }); // POST 请求
router.put('/:id', ...);   // PUT 修改
router.delete('/:id', ...); // DELETE 删除
```

- **方法**：GET（查）、POST（增）、PUT（改）、DELETE（删）
- **路径**：如 `/list`、`/register`，带动态参数就用 `:id`

### 2. 处理函数里的 req 和 res

| 变量 | 含义 | 常用属性 |
|------|------|----------|
| `req` | 请求对象 | `req.body`（POST 的 JSON）、`req.params`（URL 动态参数）、`req.query`（?key=value 查询参数） |
| `res` | 响应对象 | `res.json()` 返回 JSON、`res.status()` 设状态码 |

## 三、常用函数

### 1. 从请求里取数据

```javascript
// POST 请求体 JSON → req.body
const { username, password } = req.body;

// URL 动态参数 /api/goods/1 → req.params.id 就是 '1'
const id = req.params.id;

// URL 查询参数 /api/goods?category=手机 → req.query.category
const category = req.query.category;
```

### 2. 返回响应

```javascript
// 成功
res.json({
  code: 0,
  message: '注册成功',
  data: { id: 1, username: 'test' }
});

// 失败（带状态码）
res.status(400).json({ code: 1, message: '用户名已存在' });
res.status(404).json({ code: 1, message: '商品不存在' });
res.status(500).json({ code: 1, message: '服务器错误' });
```

### 3. 数据库操作

```javascript
// 查一条
const user = await User.findOne({ where: { username } });
const goods = await Goods.findByPk(id);

// 查多条
const list = await Goods.findAll({ order: [['id', 'ASC']] });

// 创建
const newUser = await User.create({ username, password: hashedPassword });
```

## 四、设计接口时要注意啥

### 1. 入参校验

```javascript
if (!username || !password) {
  return res.status(400).json({
    code: 1,
    message: '用户名和密码不能为空'
  });
}
```

必填项、格式等先校验，不合格直接返回，别继续执行。

### 2. 统一响应格式

和前端约定一种格式，例如：

```json
{
  "code": 0,
  "message": "成功/失败原因",
  "data": { }
}
```

- `code: 0` 成功，`code: 1` 失败
- `message` 给人看的提示
- `data` 业务数据（成功时有）

### 3. 状态码要合理

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 4. 安全

- **密码别存明文**：用 bcrypt 加密再入库
- **别把密码返回给前端**：登录成功后返回用户信息时，删掉 `password` 字段

```javascript
const userInfo = user.toJSON();
delete userInfo.password;
```

### 5. 异常要捕获

```javascript
router.post('/register', async (req, res) => {
  try {
    // 业务逻辑...
    res.json({ code: 0, ... });
  } catch (err) {
    console.error('注册失败', err);
    res.status(500).json({ code: 1, message: '注册失败，请稍后重试' });
  }
});
```

避免未捕获的异常把整个进程搞崩。

### 6. 逻辑顺序要清晰

建议顺序：**校验 → 查库/写库 → 返回**。按这个来写，代码好读好维护。

---

## 五、总结：设计 checklist

| 项目 | 说明 |
|------|------|
| 方法 | GET 查、POST 增、PUT 改、DELETE 删 |
| 路径 | 语义化，如 `/register`、`/list`、`/:id` |
| 参数 | body / params / query 分清 |
| 校验 | 必填、格式等先校验 |
| 返回 | 统一结构 `{ code, message, data }` |
| 状态码 | 200 / 400 / 404 / 500 别乱用 |
| 安全 | 密码加密、敏感字段不返回 |
| 异常 | try-catch 兜底，统一返回 500 |
