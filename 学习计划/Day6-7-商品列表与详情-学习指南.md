# 第 6~7 天：商品列表与详情 - 任务清单与学习指南

> 目标：能浏览商品、搜索、点进详情、加入购物车。  
> 本文档面向基础较弱的同学，会说明「要做啥」「怎么做」「需要会啥」。

---

## 一、任务清单（要做啥）

### 任务 1：商品列表页

- **请求后端列表接口**，拿到商品数组。
- 用 **`v-for`** 把每条商品渲染成卡片或列表项（图、名称、价格等）。
- 做 **分页** 或 **「加载更多」**：数据多时不要一次全拉，用分页或点击「加载更多」再请求下一页。

### 任务 2：搜索、排序、筛选

- 和 **后端接口约定好参数**（如关键词、排序方式），前端在请求时 **把参数带上** 即可。
- 建议先做：**关键词搜索**、**按价格排序**（升序/降序）。  
  若后端暂时没有这些参数，可以先把输入框、排序按钮做好，等后端支持后再接上。

### 任务 3：商品详情页

- 根据 **路由里的商品 id**（如 `/goods/123`）去 **请求详情接口**。
- 把接口返回的 **商品信息展示出来**（名称、价格、图片、描述、库存等）。
- 做一个 **「加入购物车」按钮**，点击后 **调用后端的加入购物车接口**（需要登录态）。

## 二、后端接口说明（当前项目已有的）

做前端前先搞清楚「后端给了什么接口、返回什么数据」。

| 接口 | 方法 | 说明 | 前端要传/要用的 |
|:-----|------|------|------------------|
| 商品列表 | `GET /api/goods/list` | 分页列表 | 查询参数：`limit`（每页条数）、`offset`（跳过条数）。返回：`{ code: 0, data: [...], total: 总条数 }` |
| 商品详情 | `GET /api/goods/:id` | 根据 id 查一条 | 路径里的 `id`（如 `/api/goods/1`）。返回：`{ code: 0, data: { id, name, price, image, description, stock, ... } }` |

- **搜索 / 排序**：若后端已支持，一般会在列表接口上加查询参数，例如 `keyword`、`sort=price_asc` 等，文档里会写；若没有，可先做前端 UI，后续再接。
- **加入购物车**：需要后端提供接口（如 `POST /api/cart/add`，传 `goodsId`、`quantity`），并需要带登录 token。若后端暂无，可先做按钮和「请先登录」或「接口开发中」的提示。

## 三、操作步骤（怎么做）

### 步骤 1：封装商品相关 API（和 Day4-5 的 user 一样）

在 `src/api/` 下新建或扩展，例如 `goods.js`：

- **商品列表**：请求 `GET /api/goods/list`，参数 `limit`、`offset`（以及以后可能有的 `keyword`、`sort`）。
- **商品详情**：请求 `GET /api/goods/商品id`。
- **加入购物车**（若后端已有）：请求 `POST /api/cart/add`，请求体里传 `goodsId`、`quantity`，请求头里带 `Authorization: Bearer token`。

用和 `user.js` 一样的 axios 实例（baseURL、以后统一带 token 的逻辑可以一起做）。

### 步骤 2：商品列表页 `GoodsListView.vue`

1. 在 `data` 里定义：商品列表数组（如 `list`）、总条数（`total`）、当前页码或 `offset`、是否加载中（`loading`）。
2. 在 `mounted` 或 `created` 里 **调一次列表接口**，把返回的 `data` 赋给 `list`，`total` 存起来。
3. 模板里用 **`v-for="item in list"`** 循环，每个 `item` 显示：图片、名称、价格；点击整块或「查看详情」用 **`<router-link :to="'/goods/' + item.id">`** 或 **`this.$router.push('/goods/' + item.id)`** 跳到详情页。
4. **分页**：根据 `total` 和每页条数算总页数，底部放「上一页」「下一页」或页码，点击时改 `offset` 再请求一次列表。
   - 或做 **「加载更多」**：当前 `offset` 增加一页的条数，再请求，把新数据 **追加**到 `list` 后面。
5. 若有 **搜索框**：输入关键词后点「搜索」或回车，把 `keyword` 传给列表接口，`offset` 重置为 0，重新请求并替换 `list`。
6. 若有 **排序**：点「价格从低到高/从高到低」时，把排序参数传给列表接口，同样重置 `offset` 并重新请求。

### 步骤 3：搜索与排序（先做简单的）

- **关键词搜索**：一个输入框 + 按钮，把 `keyword` 作为查询参数传给列表接口（后端需支持 `keyword` 参数）。
- **价格排序**：两个按钮或下拉，传 `sort=price_asc` / `sort=price_desc` 等（具体看后端约定）。  
  若后端暂时没有，可以只做 UI，传参部分注释写「等后端支持后再打开」。

### 步骤 4：商品详情页 `GoodsDetailView.vue`

1. 从路由取 id：**`this.$route.params.id`**（或用计算属性）。
2. 在 `mounted` 里根据 id 调 **详情接口** `GET /api/goods/:id`，把返回的 `data` 存到 `data` 里（如 `detail`）。
3. 模板里展示：`detail.name`、`detail.price`、`detail.image`、`detail.description`、`detail.stock` 等；图片用 `<img :src="detail.image">`。
4. **「加入购物车」按钮**：
   - 若已登录：调加入购物车接口，传当前商品 id 和数量（如 1）；成功可提示「已加入购物车」。
   - 若未登录：可先 `this.$router.push('/login')` 或提示「请先登录」。
5. 若商品不存在（接口 404 或 code≠0），可显示「商品不存在」并可选返回列表页。

### 步骤 5：加入购物车与登录态

- 加入购物车接口通常需要 **登录**，请求头里带：`Authorization: Bearer 用户的token`（token 从 Vuex 的 `state.token` 或 localStorage 取）。
- 可在 axios 的请求拦截器里统一给所有请求加上这个 Header，这样每次调接口都会自动带 token。

## 四、需要掌握的知识点（技能）

### 1. 用 `v-for` 渲染列表

- `v-for="item in list"`：遍历 `list`，每个元素叫 `item`。
- 要显示索引可以写：`v-for="(item, index) in list"`。
- 每个循环项需要不同的 key，写：`:key="item.id"`（用唯一 id 防止列表更新时错位）。

### 2. 请求数据并在页面显示

- 在 **生命周期**里请求（如 `mounted`）：页面加载完成后发请求，拿到数据赋给 `data` 里的属性，模板里就能显示。
- 异步：接口是异步的，用 **async/await** 或 **.then()** 在「请求回来」之后再改 `list`、`total` 等。

### 3. 路由参数

- 详情页路径是 `/goods/:id`，在组件里用 **`this.$route.params.id`** 拿到当前商品 id。
- 用这个 id 去请求详情接口。

### 4. 分页或加载更多

- **分页**：`offset = (当前页 - 1) * 每页条数`，每次翻页改 `offset` 再请求，用新数据替换列表。
- **加载更多**：`offset` 增加一页的数量，再请求，把新数据 **push** 进原来的 `list`。

### 5. 和后台「约定参数」

- 搜索、排序都是：**前端在请求 URL 或 body 里带上参数，后端按参数查库并返回**。
- 先和后端（或接口文档）确认：关键词叫什么（如 `keyword`）、排序怎么传（如 `sort=price_asc`），前端按约定传即可。

### 6. 带登录态发请求

- 登录后 token 存在 Vuex 或 localStorage。
- 在 axios 里用 **请求拦截器**：每次发请求前在 Header 里加上 `Authorization: Bearer <token>`，这样加入购物车等接口就能识别用户。

## 六、遇到问题可以这样查

- **列表是空的**：看 Network 里列表接口是否请求成功、返回的 `data` 是不是数组、`limit/offset` 是否传对。
- **详情页 404 或没数据**：看请求的 URL 里 id 对不对、接口返回的 `code` 和 `data`。
- **加入购物车报 401/未登录**：检查请求头里是否带了 `Authorization: Bearer token`，以及 token 是否有效。

按上面顺序做：先列表 → 再详情 → 再搜索/排序 → 最后加入购物车，一步步来即可。
