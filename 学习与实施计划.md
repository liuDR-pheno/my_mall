# 女人街商城项目 — 技术栈与半个月实施计划

> 参考项目：[Vue-nvrenjie](https://github.com/celiu-1999/Vue-nvrenjie)  
> 技术栈：前端 Vue2 + 后端 Node/Express + 数据库 MySQL

---

## 一、你需要掌握的技术栈（按优先级）

### 1. 必会（第一周重点）


| 技术                     | 用途            | 建议学习时间 | 学习重点                                             |
| ---------------------- | ------------- | ------ | ------------------------------------------------ |
| **JavaScript (ES6)**   | 前后端通用语法       | 你应已会基础 | 箭头函数、Promise、async/await、解构、模块 import/export     |
| **Vue 2 基础**           | 前端页面与交互       | 2～3 天  | 组件、data/methods、v-model、v-for、v-if、生命周期          |
| **Vue Router**         | 页面路由（多页面跳转）   | 0.5 天  | 路由配置、`<router-link>`、`this.$router.push`         |
| **Vuex**               | 全局状态（用户、购物车等） | 1 天    | state、mutations、actions、在组件中 mapState/mapActions |
| **Node.js + Express**  | 后端接口服务        | 1～2 天  | 写 GET/POST 接口、中间件、连接数据库                          |
| **MySQL + Sequelize** | 存用户、商品、订单等    | 1 天    | Model 定义、增删改查、简单查询条件                            |


### 2. 会用即可（边做边查）


| 技术                    | 用途                       |
| --------------------- | ------------------------ |
| **Axios**             | 前端发 HTTP 请求（封装成 api 即可）  |
| **Less**              | CSS 预处理（变量、嵌套），与写 CSS 类似 |
| **Element UI 或 Vant** | 现成组件库，加速做表单、表格、弹窗        |


### 3. 可选（时间紧可后补）

- **Webpack**：用 Vue CLI 生成项目即可，不必深究配置  
- **JWT / Session**：登录态；先做简单版（如 session），再考虑 JWT

## 二、半个月时间分配（15 天）

### 第 1 天：环境与项目骨架

- 安装：Node.js、MySQL、VS Code（或 Cursor）
- 用 **Vue CLI** 创建前端项目：`vue create my_mall`
- 用 **Express 生成器** 或手写一个 `server` 目录，能跑起一个 `Hello World` 接口
- 在本地安装并启动 MySQL（或使用云数据库如阿里云 RDS、腾讯云 MySQL）

**目标**：浏览器能访问前端页面，且前端能请求到后端的一个接口并拿到数据。

---

### 第 2～3 天：后端基础 + 数据库

- 设计并实现 3～5 个 **Sequelize 模型**：User、Goods、Cart、Address、Order（可先做 User、Goods）
- 写 **用户注册、登录** 接口（密码用 bcrypt 加密存储）
- 写 **商品列表、商品详情** 接口（可先写死几条数据或写个简单后台录入）

**目标**：用 Postman 能完成注册、登录、查商品。

---

### 第 4～5 天：前端登录注册与路由

- 配置 **Vue Router**：首页、登录、注册、商品列表、商品详情、购物车、个人中心等
- 做 **登录、注册页面**，表单提交后调后端接口
- 用 **Vuex** 存用户信息（token 或 user），并做简单的登录态判断（如未登录跳转登录页）

**目标**：能注册、登录，并在前端看到登录状态（如显示用户名）。

---

### 第 6～7 天：商品列表与详情

- **商品列表页**：请求后端列表接口，用 `v-for` 渲染；做分页或“加载更多”
- **搜索、排序、筛选**：对应后端接口，前端传参即可（可先做关键词搜索、价格排序）
- **商品详情页**：根据 id 请求详情接口，展示信息；**加入购物车** 按钮调后端接口

**目标**：能浏览商品、搜索、点进详情、加入购物车。

---

### 第 8～9 天：购物车

- 后端：购物车增删改查接口（与用户关联）
- 前端：**购物车页面**，展示列表、修改数量、删除、**全选与总价计算**
- 与 Vuex 配合：购物车数量同步到导航栏

**目标**：购物车可增删改，全选与总价正确。

---

### 第 10～11 天：收货地址与下单

- 后端：收货地址的增删改查、默认地址
- 前端：**地址管理页**（列表、新增、编辑、设默认）
- 后端：**创建订单** 接口（从购物车或直接购买）
- 前端：**确认订单页**（选地址、选商品、留言）、提交订单

**目标**：能选地址、提交订单并在数据库生成订单记录。

---

### 第 12～13 天：订单与个人中心

- 后端：订单列表（按状态筛选）、订单状态变更（取消、付款、收货等）
- 前端：**订单列表**（全部/待付款/待发货/待收货/待评价），订单详情、取消、支付、确认收货
- **个人中心**：个人信息修改、收藏（若时间紧可做简单版）

**目标**：订单流程跑通，个人中心能看到订单并做简单操作。

---

### 第 14 天：联调与修 bug

- 前后端联调：统一接口基地址、错误提示、登录过期处理
- 修 bug、补漏（如未登录拦截、表单校验）

---

### 第 15 天：收尾与部署（可选）

- 简单整理代码、写 README（如何启动前端、后端、数据库）
- 可选：前端 `npm run build` 打包，后端用 `static` 托管或 Nginx；MySQL 用本机或云数据库

## 四、如何用 AI 辅助（Cursor / 其它 AI）

1. **分模块让 AI 写代码**：例如“用 Express 写用户注册接口，用 Sequelize、bcrypt”、“用 Vue 写商品列表页，带分页”等，然后你自己跑通、看懂。
2. **让 AI 解释代码**：把不理解的接口或组件贴给 AI，问“这段在做什么？和登录有什么关系？”。
3. **出错时贴报错**：把终端或浏览器控制台的报错贴给 AI，让它帮你改。
4. **按接口文档开发**：先和 AI 一起定好“用户注册、登录、商品列表……”的接口 URL 和请求/响应格式，再前后端分别实现，减少联调混乱。

## 六、技术栈详解：干什么用 + 掌握什么 + 怎么学最高效

### 1. ES6（JavaScript 现代语法）

**干什么用**  
前后端都用 JavaScript 写，ES6 是「语法基础」：写变量、函数、异步、模块化都靠它。不学 ES6，看 Vue/Node 代码会一头雾水。

**要掌握什么**  
- **必会**：`const`/`let`、箭头函数、解构（对象/数组）、模板字符串、`import`/`export`  
- **必会**：`Promise`、`async/await`（调接口、等数据库结果）  
- **了解即可**：`class`、展开运算符 `...`、可选链 `?.`

**怎么学最高效**  
- 不单独开一门「ES6 课」，在写 Vue 和 Node 时**边写边查**。  
- 遇到不认识的语法（如 `async/await`）就查 MDN 或「ES6 入门」对应章节，写 2～3 个接口 + 2～3 个页面就够用了。

---

### 2. Vue 2（前端框架）

**干什么用**  
把页面拆成「组件」，用数据驱动视图（改 `data` 就自动更新 DOM）。商城里的商品列表、购物车、表单都是组件，用 Vue 写比手写 jQuery 清晰、好维护。

**要掌握什么**  
- **必会**：`data`、`methods`、`computed`；`v-model`、`v-for`、`v-if`/`v-show`；组件 `props`、`$emit`  
- **必会**：生命周期：`created`/`mounted`（在这里调接口、拿数据）  
- **会用即可**：`watch`、`$refs`、组件通信（先搞懂 props + emit）

**怎么学最高效**  
- 直接看 [Vue 2 官方文档](https://v2.cn.vuejs.org/) 的「基础」「组件」两章，**不要**从头到尾背。  
- 每看一个小节就**在项目里写一个对应功能**（例如看完 `v-for` 就写商品列表），这样 2～3 天能上手。

---

### 3. Vue Router（前端路由）

**干什么用**  
单页应用里「换页面」不刷新整页，而是根据 URL 显示不同组件（首页/登录/商品详情/购物车等）。Router 负责：URL ↔ 组件的对应关系、跳转、传参。

**要掌握什么**  
- **必会**：在 `router/index.js` 里配置 `path`、`component`、`name`  
- **必会**：`<router-link to="...">` 和 `this.$router.push()` 做跳转  
- **必会**：路由传参：`params` 或 `query`，在页面上用 `this.$route.params`/`query` 取  
- **会用即可**：路由守卫 `beforeEach`（做「未登录跳转登录页」）

**怎么学最高效**  
- 跟着官方文档「起步」配一遍路由，再在你的项目里加 5～6 个页面（首页、登录、注册、商品列表、详情、购物车），**边配边跑**，半天足够。

---

### 4. Vuex（全局状态）

**干什么用**  
多个页面都要用的数据（用户信息、购物车数量、token）放在 Vuex 里，一处改、处处更新。不用 Vuex 就要层层 `props` 或事件，很乱。

**要掌握什么**  
- **必会**：`state`（存数据）、`mutations`（同步改 state）、`actions`（异步，如调接口后 commit mutation）  
- **必会**：在组件里用 `mapState`、`mapActions` 或直接 `this.$store.state`、`this.$store.dispatch`  
- **会用即可**：`getters`（对 state 做计算）、模块化（项目大了再学）

**怎么学最高效**  
- 先搞清「为什么要 Vuex」（多页面共享用户/购物车），再跟着官方文档做一遍「状态 + 修改」的流程。  
- 在项目里先只做两件事：**存用户信息**、**存购物车数量**，做完就够用了，约 1 天。

---

### 5. Node.js + Express（后端服务）

**干什么用**  
Node 让 JavaScript 在服务器运行；Express 是写 HTTP 接口的框架。你的「注册、登录、商品列表、下单」都是 Express 提供的一个个 API，前端用 Axios 调这些 API。

**要掌握什么**  
- **必会**：`app.get()`、`app.post()` 写接口；`req.query`、`req.body`、`req.params` 拿参数  
- **必会**：中间件：`express.json()`（解析 JSON）、自己写「登录校验」中间件  
- **必会**：用 Sequelize 在路由里做增删改查（见下）  
- **会用即可**：`Router` 拆分路由、`next(err)` 统一错误处理

**怎么学最高效**  
- 不看大而全的 Node 教程，直接「用 Express 写 3 个接口」：一个 GET（如商品列表）、一个 POST（如注册）、一个要登录的 GET（如当前用户信息）。  
- 写完后用 Postman 调通，再让前端来调，1～2 天能上手。

---

### 6. MySQL + Sequelize（数据库与 ORM）

**干什么用**  
MySQL 是关系型数据库，用户、商品、订单、购物车都存这里。Sequelize 是 Node 里操作 MySQL 的 ORM 库，用来定义「表结构（Model）」和做增删改查。

**要掌握什么**  
- **必会**：用 Sequelize 定义 Model（字段、类型、是否必填、表名）  
- **必会**：`Model.create()`、`Model.findAll()`、`Model.findByPk()`、`Model.update()`、`Model.destroy()`  
- **必会**：简单查询：`where`、`order`、`offset`、`limit`（列表分页）  
- **会用即可**：关联 `belongsTo`/`hasMany`（如订单里关联用户、商品）、`include` 做联表查询

**怎么学最高效**  
- 先在本机或云上起一个 MySQL，用 Sequelize 连上。  
- 只做两件事：定义一个 User 模型并插入一条；写一个「根据 id 查用户」的接口。做完再扩展到 Goods、Order，1 天够用。

---

### 7. Axios（HTTP 请求）

**干什么用**  
前端用 Axios 发 GET/POST 请求到你的 Express 接口，拿到 JSON 数据再渲染到页面。不用 Axios 就要用原生 `fetch` 或 `XMLHttpRequest`，写法麻烦。

**要掌握什么**  
- **必会**：`axios.get(url)`、`axios.post(url, data)`；用 `async/await` 或 `.then()` 处理结果  
- **必会**：封装：`axios.create({ baseURL })`，统一写 `api/user.js`、`api/goods.js`，组件里只调 `api.xxx()`  
- **会用即可**：请求/响应拦截器（统一加 token、统一处理 401 跳登录）

**怎么学最高效**  
- 不单独学，在「第 4～5 天」做登录注册时，让前端调一次 `axios.post('/api/register', user)`，再封装成 `api/register(user)`，后面所有接口都按这个模式，边做边会。

## 七、小结

- **必会**：Vue 2 + Vue Router + Vuex、Node + Express、MySQL + Sequelize、ES6、Axios。
- **策略**：前 3 天搭好架子 + 跑通注册登录与商品接口；中间 7 天做商品、购物车、地址、订单；最后 2 天联调与修 bug。
- **用 AI**：按“接口 → 后端实现 → 前端调用”一步步来，每步都自己运行、理解，半个月完成一个可演示的商城是可行的。
- **学习心法**：每个技术都「先知道干什么用 → 在项目里用一次 → 再查文档补细节」，比系统啃完再写项目高效得多。
